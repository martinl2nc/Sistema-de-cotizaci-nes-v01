# 📄 DOC 1: Esquema de Base de Datos (ERD) - Supabase / PostgreSQL

## 1. Reglas de Diseño Aplicadas

* **Identificadores Mixtos:** Usaremos `UUID` (códigos alfanuméricos seguros) para que las máquinas (n8n, Aplicación React, Webhooks) se comuniquen sin errores. Pero añadiremos un campo `numero_correlativo` autoincremental para generar tu código `COT-0001`.
* **Flexibilidad de Productos:** En la tabla de las líneas de cotización, el enlace al Catálogo (`producto_id`) será **Opcional (Nullable)**. Esto permite que el vendedor escriba un producto personalizado que no existe en la base de datos sin que el sistema lance un error.
* **Preparado para el Futuro:** Se han añadido las columnas `woo_order_id` (en cotizaciones) y `woo_product_id` (en productos) para que en una fase 2 puedas sincronizar tu catálogo bidireccionalmente.
* **Protección de Datos:** Las llaves foráneas están configuradas con `ON DELETE RESTRICT` para evitar borrar clientes/productos por error, y con `ON DELETE CASCADE` para las líneas de cotización (si borras una cotización, se borran sus detalles automáticamente para no dejar basura).

---

## 2. Diccionario de Tablas (Para tu documentación)

1. **`empresa_configuracion`**: Guarda los datos de tu empresa (RUC, Cuentas bancarias) para imprimirlos en el PDF.
2. **`vendedores`**: Lista de tu equipo comercial.
3. **`categorias`**: Para clasificar el catálogo.
4. **`productos`**: Tu catálogo maestro. Precios en Soles (PEN).
5. **`clientes`**: Base de datos de clientes adaptada para B2B y B2C (RUC/DNI, Razón Social de la empresa, Nombres y Apellidos del contacto, Dirección).
6. **`cotizaciones`**: Cabecera del documento. Contiene el subtotal, descuento global, IGV, y las observaciones para el PDF.
7. **`cotizaciones_lineas`**: El detalle. Guarda qué se cotizó, cantidad, precio histórico y subtotal de esa línea.

## 3. Script SQL de Implementación (Copiar y Pegar en Supabase)

*Instrucciones: Ve a tu panel de Supabase > SQL Editor > New Query. Pega todo este bloque de código y presiona "Run".*

```sql
-- =========================================================================
-- SISTEMA DE GESTIÓN DE COTIZACIONES - SCRIPT DE CREACIÓN DE BASE DE DATOS
-- =========================================================================

-- 1. TABLA: Configuración de Empresa (Textos estáticos para el PDF)
CREATE TABLE empresa_configuracion (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    razon_social VARCHAR(255) NOT NULL,
    ruc VARCHAR(20) NOT NULL,
    direccion TEXT,
    cuentas_bancarias TEXT,
    terminos_condiciones TEXT
);

-- 2. TABLA: Vendedores
CREATE TABLE vendedores (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    nombre VARCHAR(255) NOT NULL,
    email VARCHAR(255) UNIQUE,
    activo BOOLEAN DEFAULT true
);

-- 3. TABLA: Categorías de Producto
CREATE TABLE categorias (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    nombre VARCHAR(100) NOT NULL
);

-- 4. TABLA: Catálogo de Productos
CREATE TABLE productos (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    woo_product_id BIGINT UNIQUE, -- Para futura sincronización con WooCommerce
    sku VARCHAR(50) UNIQUE,
    nombre VARCHAR(255) NOT NULL,
    descripcion TEXT,
    categoria_id UUID REFERENCES categorias(id) ON DELETE SET NULL,
    precio_base NUMERIC(10, 2) NOT NULL DEFAULT 0.00, -- Precios en Soles
    activo BOOLEAN DEFAULT true,
    fecha_creacion TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc', now())
);

-- 5. TABLA: Clientes (Actualizada B2B / B2C)
CREATE TABLE clientes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tipo_documento VARCHAR(20) DEFAULT 'RUC', -- RUC o DNI
    numero_documento VARCHAR(20) UNIQUE,
    razon_social VARCHAR(255), -- Puede estar vacío si es un cliente DNI (Persona Natural)
    nombres_contacto VARCHAR(100) NOT NULL, -- Obligatorio (Viene del First Name de Woo)
    apellidos_contacto VARCHAR(100) NOT NULL, -- Obligatorio (Viene del Last Name de Woo)
    email VARCHAR(255) NOT NULL,
    telefono VARCHAR(50),
    direccion TEXT,
    comprobante_preferido VARCHAR(20) DEFAULT 'Factura',
    fecha_creacion TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc', now())
);

-- 6. TABLA: Cotizaciones (Cabecera)
CREATE TABLE cotizaciones (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    numero_correlativo BIGINT GENERATED BY DEFAULT AS IDENTITY, -- Genera 1, 2, 3... (En la Aplicación React le pondremos "COT-" adelante)
    origen VARCHAR(50) DEFAULT 'Manual', -- 'Web' o 'Manual'
    woo_order_id BIGINT UNIQUE, -- ID del borrador que viene de WooCommerce
    cliente_id UUID REFERENCES clientes(id) ON DELETE RESTRICT NOT NULL,
    vendedor_id UUID REFERENCES vendedores(id) ON DELETE RESTRICT,
    
    fecha_emision DATE DEFAULT CURRENT_DATE,
    fecha_validez DATE DEFAULT CURRENT_DATE + 15, -- Válido por 15 días por defecto
    estado VARCHAR(50) DEFAULT 'Por Revisar', -- 'Por Revisar', 'Aprobada', 'Generar PDF', 'Enviada', 'Cancelada'
    observaciones_pdf TEXT, -- Notas dinámicas que irán impresas en el PDF
    
    aplica_igv BOOLEAN DEFAULT true,
    subtotal NUMERIC(10, 2) DEFAULT 0.00,
    descuento_global_monto NUMERIC(10, 2) DEFAULT 0.00,
    igv_monto NUMERIC(10, 2) DEFAULT 0.00,
    total_final NUMERIC(10, 2) DEFAULT 0.00,
    
    fecha_creacion TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc', now()),
    ultima_actualizacion TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc', now())
);

-- 7. TABLA: Líneas de Cotización (Detalle)
CREATE TABLE cotizaciones_lineas (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    cotizacion_id UUID REFERENCES cotizaciones(id) ON DELETE CASCADE NOT NULL, -- Si se borra la cotización, se borran las líneas
    producto_id UUID REFERENCES productos(id) ON DELETE RESTRICT, -- Opcional: Puede ser NULL si es un producto personalizado
    
    nombre_producto_historico VARCHAR(255) NOT NULL, -- Obligatorio: Guarda el nombre exacto al momento de la venta
    cantidad INTEGER NOT NULL DEFAULT 1,
    precio_unitario NUMERIC(10, 2) NOT NULL DEFAULT 0.00, -- Precio antes de descuentos de línea
    descuento_linea_monto NUMERIC(10, 2) DEFAULT 0.00, -- Si le das descuento a este producto en específico
    subtotal_linea NUMERIC(10, 2) NOT NULL DEFAULT 0.00 -- (cantidad * precio_unitario) - descuento
);

-- =========================================================================
-- MEJORAS DE INFRAESTRUCTURA Y RENDIMIENTO (SUPABASE BEST PRACTICES)
-- =========================================================================

-- A. SEGURIDAD: Habilitar Row-Level Security (RLS) en todas las tablas
ALTER TABLE empresa_configuracion ENABLE ROW LEVEL SECURITY;
ALTER TABLE vendedores ENABLE ROW LEVEL SECURITY;
ALTER TABLE categorias ENABLE ROW LEVEL SECURITY;
ALTER TABLE productos ENABLE ROW LEVEL SECURITY;
ALTER TABLE clientes ENABLE ROW LEVEL SECURITY;
ALTER TABLE cotizaciones ENABLE ROW LEVEL SECURITY;
ALTER TABLE cotizaciones_lineas ENABLE ROW LEVEL SECURITY;

-- A.1 Políticas Base para la Aplicación React 
-- (n8n usa la Service Role Key, por lo que ignora estas reglas y tiene acceso total)
CREATE POLICY "Lectura pública de config" ON empresa_configuracion FOR SELECT USING (true);
CREATE POLICY "Lectura pública de vendedores" ON vendedores FOR SELECT USING (true);
CREATE POLICY "Lectura pública de categorias" ON categorias FOR SELECT USING (true);
CREATE POLICY "Lectura pública de productos" ON productos FOR SELECT USING (true);

-- POLÍTICAS TEMPORALES PARA DESARROLLO (Frontend React)
-- Todo el acceso está permitido sin login. ANTES DE PRODUCCIÓN cambiar "true" por "auth.role() = 'authenticated'"
CREATE POLICY "Acceso total a clientes en Desarrollo" ON clientes FOR ALL USING (true);
CREATE POLICY "Acceso total a cotizaciones en Desarrollo" ON cotizaciones FOR ALL USING (true);
CREATE POLICY "Acceso total a lineas en Desarrollo" ON cotizaciones_lineas FOR ALL USING (true);

-- B. RENDIMIENTO: Creación de índices en Llaves Foráneas (Evita 'Full Table Scans')
CREATE INDEX idx_productos_categoria ON productos(categoria_id);
CREATE INDEX idx_cotizaciones_cliente ON cotizaciones(cliente_id);
CREATE INDEX idx_cotizaciones_vendedor ON cotizaciones(vendedor_id);
CREATE INDEX idx_lineas_cotizacion ON cotizaciones_lineas(cotizacion_id);
CREATE INDEX idx_lineas_producto ON cotizaciones_lineas(producto_id);

-- C. INTEGRIDAD: Triggers para actualización automática de "ultima_actualizacion"
-- C.1 Instalar extensión de Postgres
CREATE EXTENSION IF NOT EXISTS moddatetime schema extensions;

-- C.2 Crear el Trigger para la tabla principal
CREATE TRIGGER handle_updated_at BEFORE UPDATE ON cotizaciones
  FOR EACH ROW EXECUTE PROCEDURE moddatetime (ultima_actualizacion);
```

### ¿Qué logramos con este diseño?

1. **Escalabilidad Total:** Fíjate en la columna `woo_product_id` en productos y `woo_order_id` en cotizaciones. Cuando quieras administrar todo desde tu sistema, n8n sabrá exactamente qué producto actualizar en la web.
2. **Productos Personalizados (Caso #3 resuelto):** Como `producto_id` dice `Opcional / Puede ser NULL`, tu vendedor puede abrir la Aplicación React, dejar el selector de productos en blanco, escribir "Instalación de Software Especial" en la casilla `nombre_producto_historico`, ponerle 500 Soles, y el sistema lo guardará perfectamente sin alterar el catálogo.
3. **Código COT-0001 (Caso #4 resuelto):** Tienes la columna `numero_correlativo` que genera 1, 2, 3 automáticamente sin fallar jamás. Luego, en la Aplicación React y en tu PDF, simplemente le diremos que muestre `"COT-" + numero_correlativo` (Ej: COT-1). Si quieres rellenarlo con ceros (COT-0001), APITemplate y React lo hacen con una simple configuración visual.
4. **Clientes Flexibles B2B/B2C:** Al tener `razon_social` opcional, y `nombres_contacto` obligatorio, el sistema se adapta orgánicamente a los datos que envía el carrito de compras de WooCommerce, evitando conflictos de inserción y permitiendo emitir PDFs tanto a empresas como a personas naturales.
